# Project

Behavioral Financial Wallet (Smart Wallet)

---

# Why This Project Exists

Every software project begins with a problem.

For me, that problem was not a lack of budgeting applications—it was the lack of self-control when spending digital money.

As a student, I receive a fixed monthly allowance and always begin the month with the intention of saving a portion of it. However, with the convenience of UPI and digital payments, spending money has become almost effortless. A purchase now takes only a few taps, and because money is no longer physical, it is much harder to notice it disappearing.

I often found myself checking my account balance repeatedly, mentally calculating how much I had already spent, how much I wanted to save, and whether I could afford another purchase. More often than I would like, the money I had planned to save slowly disappeared through small, impulsive expenses.

This project was born from that experience.

Rather than creating another application that simply records transactions, I wanted to build something that encourages thoughtful financial decisions before the money is spent.

---

# Vision

The goal of the Smart Wallet is to help users build healthier saving habits while ensuring they remain in complete control of their own money.

Instead of restricting users or making financial decisions on their behalf, the application acts as a commitment tool. Users voluntarily lock money for personal goals, separate savings from spending money, and receive gentle reminders that encourage mindful spending rather than enforcing limitations.

The application is designed around one simple philosophy:

> Protect future versions of yourself from today's impulsive decisions.

---

# Core Principles

Throughout the development of this project, every feature should support one or more of the following principles.

### 1. User First

The application should be simple enough for a first-time user to understand without needing lengthy instructions.

### 2. Freedom Over Restriction

The user's money always belongs to the user. The application should encourage discipline without removing financial freedom.

### 3. Behavioral Finance

Every major feature should help users develop healthier spending and saving habits rather than simply displaying financial information.

### 4. Purposeful Saving

Savings should always have meaning. Users should be encouraged to create named goals that provide motivation and a sense of achievement.

### 5. Transparency

Users should always know:

- Their total balance.
- Their available spending balance.
- Their locked savings.

Financial information should never feel confusing or hidden.

### 6. Learn Before Building

Every line of code added to this project should be fully understood before becoming part of the application.

Understanding is more valuable than completing features quickly.
